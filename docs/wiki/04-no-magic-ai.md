# No Magic AI

No Magic means every AI answer should make it clear what came from data, what came from deterministic logic, and what the AI interpreted or drafted.

## Three Layers

1. Observed data: Brand Health Record metrics, source files, trends, occasions, and GN bridge signals.
2. Deterministic logic: diagnosis rules, evidence ledger, treatment links, ranking logic, and guardrails.
3. AI interpretation: plain-English summary, translation, challenge, narrative, and meeting-ready framing.

AI should explain and interrogate. It should not invent facts, diagnoses, treatments, competitors, causality, SKU pricing recommendations, cannibalization proof, or portfolio migration proof.

## Chat Scope

Dialog With Data is scoped to:

- Active brand.
- Active visual.
- Evidence ledger.
- Rule trace.
- Treatment library.
- Follow-up signals.
- Selected persona.

If a question needs data outside the packet, the assistant should say what is missing.

## Why This Matters

Stakeholders will be skeptical because the technology is new and the output is polished. The system should make skepticism productive by exposing citations, rule traces, and caveats in context.

## Current Surfaces

- Report sidecar chat renders markdown and scoped answers.
- Brand conversation page includes expandable No Magic citations.
- Live Consult attaches evidence chips to transcript moments.
- Data view exposes the source packet.
