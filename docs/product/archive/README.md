# Product Documentation Archive

This archive preserves product plans, PRDs, and strategy notes that shaped the prototype but are no longer the active execution path.

Archive rules:

- Keep historical docs intact when possible.
- Add a dated folder README explaining why documents moved.
- Update active docs to point to current plans.
- Do not delete historical docs unless they are duplicates with no unique context.
- Archived docs can still be cited as rationale, but active build decisions should come from `docs/product/README.md`, `PLANS.md`, current PRDs/plans, `STATUS.md`, and `BACKLOG.md`.
