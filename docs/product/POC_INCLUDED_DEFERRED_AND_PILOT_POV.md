# POC Included, Deferred, And Pilot POV

## Purpose

This note is the current meeting-ready summary of what the BBE Brand Doctor POC includes, what is intentionally deferred, and what a real pilot/production project would need.

The POC target is fundable proof, not production architecture. The system should feel dynamic, intelligent, evidence-grounded, and easy to update through data, knowledge, rules, treatments, instructions, and approved output skills.

## Included In The Current POC

### Product Foundation

- One brand home, one intelligence agent, many durable outputs.
- Brand-scoped data and knowledge served through `BrandHealthRecord` and `BrandIntelligencePacket`.
- Local JSON-first data, rules, treatments, prompts, output standards, dynamic views, and artifact models.
- Global-vs-brand separation: treatment definitions are global; treatment recommendations are ranked and explained through the active brand's diagnosis, evidence, and caveats.
- Current v7 product truth: Perceived Value user language, SMD order, Momentum as verdict, Ahead/Behind as size-check, Room to Grow as provocation context, peer basis, trend caveats, and output quality checks.

### Agent And Jarvis Foundation

- Stable Assistant and premium Jarvis Preview share the same governed brain.
- Direct answers stay conversational and answer-first.
- Explicit build/create/draft/package asks become approval-gated governed work.
- Jarvis can create durable Brand Work Items and expose them through `/brand/[brandId]/work`.
- Recent work persists locally for the POC; enterprise persistence remains gated.
- Voice is user-initiated and POC-bounded, with current routing into the same Assistant/Jarvis brain.

### Current Data And Knowledge Coverage

- Best-data POC brands: Lay's, Cheetos, Siete, and Tostitos.
- Priority brands include prototype/reviewed-public Brand Strategic Context packets.
- Priority brands include POC Momentum source packets with peer set, market context, Room to Grow inputs, and SMD contribution weights.
- Codex-created and Codex-assisted assumptions are cataloged separately from user/source-provided inputs.
- Evidence, gaps, source readiness, and caveats are visible instead of hidden inside agent prose.

### Core Output Skills

- Executive QBR / CMO Review Read: verdict, leadership takeaways, BBE bloodwork, proof cards, guardrails, inline approved views, treatment bridge, next decision/test path, and governance disclosure.
- Treatment Recommendation / Treatment Read: diagnosis-to-treatment bridge, ranked paths to consider, brand-specific basis, evidence to inspect, guardrails, approved evidence views, and next test path.
- Evidence Read / Proof Pack: actual data basis, metric cards, proof cards, source posture, gaps and caveats, guardrails, approved evidence views, next proof path, and metadata disclosure.
- Assumption/Readiness Read: composition routing exists and should be the next artifact-quality pass.
- Brief / Story Builder remains part of the approved future skill direction but should wait until the three core decision artifacts plus readiness are excellent.

## Deferred And Why

### Official Export, Circulation, And Publishing

Deferred to pilot/production because the artifact needs reviewer identity, access control, versioning, retention, and official approval policy before it can circulate as a governed business output.

### Enterprise Persistence

Deferred because the POC only needs local durability to prove the work object model. A real pilot needs database-backed persistence, identity, audit, retention, backup/recovery, and environment controls.

### Canonical Source Writes

Deferred because the system should not write source truth until source-owner governance, validation workflows, review authority, rollback, and lineage are approved.

### Source-Owner Replacement Of POC Assumptions

Deferred from the POC build but required for pilot. The POC can use clearly labeled assumptions to prove system behavior; pilot needs approved competitive sets, measured Room to Grow, contribution weights, trend significance, and official Brand Strategic Context.

### Treatment Outcome Learning And Efficacy Claims

Deferred because treatment outcomes require baseline/follow-up methodology, accepted outcome records, review, and enough post-action data. The POC should recommend paths to consider, not claim proven treatment efficacy.

### Detailed Project Planning

Deferred by design. Current treatment outputs should recommend treatment paths and areas to inspect, not owner/date/task-level execution plans.

### Audience/Persona Variants And Artifact Recomposition

Deferred until core skill quality is strong. The architecture can support CMO, insights, operator, agency, and brand-lead variants later, but funding value depends first on excellent base artifacts.

### Arbitrary Generated UI

Deferred indefinitely unless governance changes. The product should compose from approved modules and dynamic views, not generate unconstrained React/HTML.

## Real Pilot / Production POV

### Data Integrations

A real project needs governed feeds or source-owner file drops for BBE tracker data, Growth Navigator, peer/competitive set definitions, category and market context, Room to Grow, trend/significance, Brand Strategic Context, creative/activation context, and eventually outcome measures. Each feed needs freshness, source period, lineage, owner, validation rules, and data quality reporting.

### Knowledge Layer

The knowledge layer should include brand strategy packets, a treatment ontology, diagnosis rules, PepsiCo POV, guardrails, source catalogs, assumption catalogs, output policies, and reusable audience/output semantics. This is the "brain" marketers can update without rewriting UI.

### Intelligence Layer

The strongest architecture is hybrid: deterministic rules and services compute diagnoses, metric reads, gaps, guardrails, and treatment ranking; retrieval and LLMs explain, translate, challenge, summarize, and assemble governed work. The agent should be trained through curated examples, evals, and human feedback, not uncontrolled self-learning.

### Artifact And Workflow Layer

Pilot needs approved artifact templates/modules, work item versioning, comments, review states, approvals, export/circulation gates, and artifact-level metadata. The work object is the business unit of value, not the chat transcript.

### Governance, Security, And Compliance

Production requires identity, role-based access, audit logs, privacy/retention policy, source promotion gates, export controls, review identity, and clear separation of prototype assumptions from approved business evidence.

### Evaluation And Operating Model

The team will need golden test cases, adversarial tests, brand-by-brand acceptance checks, artifact quality reviews, source freshness checks, and stakeholder UAT. Success should be judged on trust, specificity, executive usefulness, decision clarity, and time saved, not only model accuracy.

### Rollout Work

A credible pilot should start with a small number of priority brands, source-owner sponsorship, named users across CMO/Insights/brand/operators, training materials, support workflow, and a clear funding ask for data integration, governance, artifact workflow, and intelligence tuning.
