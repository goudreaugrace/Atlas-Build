# ATLAS Lightweight Intelligence UI Direction

This is the visual direction for ATLAS going forward.

## Product Feel

ATLAS should feel like a premium intelligence briefing and working report, not a dashboard, slide editor, or generic AI app. The UI should help a European CNO quickly understand what matters, what changed, where money is at risk, and where to drill next.

## Core Principles

- Lead with the read: one strong summary statement before supporting details.
- Use white space, typography, and alignment before adding containers.
- Use cards only for distinct objects: a buyer, a metric read, a timeline item, a document, a generated output, or a critical risk state.
- Avoid dashboard grids, nested panels, heavy borders, colored frames, and repeated explanatory copy.
- Financial impact should be visible and easy to scan, with restrained conditional color only on the numbers that need attention.
- Sources should be visible but quiet: source name, date, confidence, and status should support trust without competing with the main read.
- Navigation should feel calm and report-like: compact tabs, clear links, and direct drilldowns.

## Visual Language

- Background: white or near-white only.
- Typography: navy/ink headings, compact labels, strong financial numerals.
- Accent: PepsiCo blue for links, selected states, and primary actions.
- Risk: red only for true risk numbers or high-risk states.
- Warning: amber only for watch items or medium gaps.
- Success: green only for favorable movement or accepted/approved states.
- Containers: soft radius, subtle border, minimal shadow, no glow.
- Motion: light hover only, never decorative animation.

## Pattern Rules

- Header read modules can use a large rounded white card.
- Lower page areas should remain an open canvas unless the content object needs containment.
- Snapshot cards should be compact and comparable, with the most important number or status first.
- List cards should be scannable at a glance: buyer name, why it matters, key metrics, and drilldown affordance.
- Generated or AI-created views should look like report sections, not chatbot responses.
- Ask ATLAS remains a small sticky command surface, not a page-dominating chat panel.

## Anti-Patterns

- Full-page blue backgrounds.
- Boxed sections inside boxed sections.
- Decorative AI visual effects.
- Cards used only to fill space.
- Long explanatory headers above every section.
- Tables or grids as the first read when a summary would answer the user’s question faster.
