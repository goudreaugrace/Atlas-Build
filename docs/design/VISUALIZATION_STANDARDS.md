# Visualization Standards

## Visualization grammar

### KPI + benchmark chips

Use for DP, PP, M, D, S. Each metric should show:
- value
- vs category band
- Ahead status
- Momentum
- one-line interpretation

### Sparkline trend

Use for quarterly movement. Do not overstate short-term movement when BBE is expected to move slowly.

### Root cause map

Use a driver relationship diagram:

```text
Meaningful + Salient → Demand Power
Meaningful + Different -> Perceived Value
```

Use careful language: “typically read through,” “consistent with,” or “diagnostic pathway,” not “caused by” unless causal proof exists.

### BrandZ map

Use typology grid to show brand shape. Use plain-English typology descriptions.

### GN bridge tree

Use for commercial conversion:
- Proposition
- Reach
- Resonance
- Available & Visible
- Delivering Value

### Treatment pathway

Use for prescription sequencing:

```text
Foundation → Acceleration → Specialist → Follow-up
```

### Evidence drawer

Use tabs or grouped rows:
- Supporting
- Complicating
- Missing
- Source periods

## Dialog with visuals

Every visual should expose suggested questions. Chat should know which visual is active.
