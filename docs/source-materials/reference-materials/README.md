# Reference Materials

This folder is the drop zone for third-party, partner, and stakeholder reference material that may inform Brand Doctor learning content, data design, or future product concepts.

## Where To Put New Files

Use this structure:

- `inbox/`
  - Put unsorted files here when speed matters.
  - Use a dated folder if there is more than one file, for example `inbox/2026-06-16-ali-dump/`.
- `ali-dumps/YYYY-MM-DD-short-topic/`
  - Use this for bundles from Ali or other stakeholder dumps once the theme is clear.
- `external-frameworks/YYYY-MM-DD-short-topic/`
  - Use this for named third-party frameworks, published papers, or outside methodology references.
- `source-reports/YYYY-MM-DD-short-topic/`
  - Use this for actual report exports, screenshots, PDFs, decks, and tables.
  - Prefer a domain folder first when the theme is known, for example `source-reports/growth-navigator/2026-06-16/`.

If you are not sure, put the files in `inbox/` and leave the file names descriptive. The manifest can be updated after triage.

## Use Rules

These materials are not approved product copy by default.

Use them as:

- evidence of concepts the prototype should teach,
- inspiration for education modules,
- source context for data-design decisions,
- examples of useful diagnostic patterns.

Do not use them as:

- proof of PepsiCo-approved methodology,
- source facts for a brand diagnosis,
- official endorsement by third parties,
- final language for user-facing claims without review.
